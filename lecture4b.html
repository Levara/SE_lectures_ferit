Entity--Relationship modeling

  • ERM razdvaja logičku, semantičku strukturu od implementacije
    • entity distinguishable object of some type
    entity type type of a set of entities
    attribute value piece of information describing an entity
    attribute type of a set of attribute values
    relationship association betwee two or more entities

Nešto o objektima

  • Svrha dizajna i analize:
  • Modeliranje objekata u stvarnom svijetu (Europski pogled)
    vs.
  • Identifikacija iskoristivih komponenti i izgradnja hijerarhije naslijeđivanja (Američki pogled)
  • Što je objekt? -- više različitih stajališta
  • object = identity + variables + operations
  • object = identity + state + behaviour

Objekt sa stajališta modeliranja

  • Konceptualni model nekog dijela stvarnog ili imaginarnog svijeta
  • Europski pogled
  • Karakteristike:
    1. Svaki objekt ima identitet
    2. Svaki objekt ima svojstva
  • Promjenjivo stanje + sučelje

Objekt sa filozofskog stajališta

  • "Sve je objekt" stajalište
  • Dva različita tipa apstrakcija:
    • Entiteti s prirodnim početkom i krajem
    • "Vječni" nepromjenjivi entiteti (values)

Objekt sa stajališta PI

  • Apstrakcije podataka
  • Enkapsuliraju i podatke i operacije
  • Object-based jezici
  • Object-oriented jezici (nasljeđivanje)

Objekt sa stajališta implementacije

  • Kontinuirana struktura u memoriji
  • Kompozitni objekti (sadrže manje pod-objekte)
  • Najniža razina objeka su 'atomski' podaci

Objekti

  • Karakterizira ih set atributa (svojstava)
    • Intrinsična svojstva (boja, visina i slično)
    • Koncept atributa potječe iz ERM-a
    • Definira stanje (state) objekta
  • Veze (Relationships) označavaju zajednička svojstva više objekata
  • Objekti s istim setom atributa pripadaju istoj klasi
  • Enkapsulira i stanje i ponašanje
  • Primjer: Customer(Name, Address, BooksOnLoan)
    • Instanciranje, uništavanje, servisi

UML

  • Započeo 1980-ih
  • Aktualna verzija: UML2
  • Najčešće korištena notacija za inženjering zahtjeva i dizajn
  • Statički i dinamički dijagrami

Tipovi UML dijagrama

Dijagram Opis
Activity (D) Model business processes, workflow, procedural logic. Flowcharts + paralelism
Class (S) Model classes, their features and relationships
Communication (D) Model flow of messages between instances of classes
Component (S) Model a set of components and their interrelationships (through interfaces)
Composite Structure Model dynamic structure of a class
Deployment Model physical layout, i.e. assignment of system elements to hardware
Interaction overview (S) Combines activity diagrams and sequence diagrams
Object (S) Model objects at some point in time, aka. instance diagram
Package (S) Model grouping of elements into packets
Sequence (D) Model order of messages between instances of classes
State machine (D) Model states in which an object can be, and transitions between states
Timing (D) Model state changes of an object over time
Use case (D) Model use cases
